# Visual Copywriter Session Context

本文档记录了项目在开发过程中的核心规范、待办任务及技术债务，以便后续接手时保持设计语言与代码逻辑的高度一致。

## 核心开发规范 (Core Development Norms)

### 1. 审美与设计语言
- **极简主义原则**：禁止使用刺眼的红色提示（如报错感叹号、红色按钮）。错误状态应以低饱和度或中性色（如淡灰色）呈现，保持“文青”的高级感。
- **字体规范**：西文使用 `DM Sans` 或 `Playfair Display`，中文主要使用 `LXGW WenKai (落霞文楷)` 以增强文学气息。
- **动态标签**：AI 标签应根据模式展示为 `✦ [Model] 瞎编`（原创）或 `✦ [Model] 搬运`（引文），带有微型的 `✦` 符号。

### 2. 交互逻辑规范
- **任务独立受控**：支持单张图片的“停止生成”与“重新生成”。点击“停止”时**禁止移除**图片卡片。
- **移除即销毁**：右上角的“X”按钮是移除图片的唯一入口。
- **默认配置**：为了降低认知负担，前端隐藏了模型选择器。默认模式设为“**经典引文匹配**”，默认模型锁定为 **Gemini 3 Flash**。
- **上传限制**：全局限制单次任务最多 **6 张图片**。超出部分必须自动截取，并伴随优雅的 Toast 提示。

### 3. 数据清洗与导出
- **反“幻觉”清洗**：所有大模型返回的文案必须通过 `cleanAIResponse` 函数，严禁泄露内部思考过程、Markdown 符号或 JSON 包装层。
- **导出优化**：前端预览应与 Canvas 导出效果保持高度对齐。

### 4. 本地开发规范 (NEW)
- **高性能编译**：默认开启 Next.js Turbopack (`next dev --turbo`) 以保持亚秒级的冷启动速度。
- **环境隔离**：确保 NVM 初始化脚本已持久化至 `~/.zshrc`，避免 shell 环境丢失 `node/npm`。

## 技术债务 (Identified Technical Debt)

### 1. 状态管理
- `useImageProcessor.ts` 中的 `images` 数组状态日益复杂，未来可能需要引入更现代的状态库或 `useReducer` 重构。

### 2. 导出可靠性
- `html2canvas` 方案依赖浏览器渲染，在处理某些极长引文时的折行效果可能与预览有微小偏差。长期方案可考虑服务端生成（如 Puppeteer）。

### 3. 内存管理
- 频繁的图片上传与生成会产生大量文件 Blob 和 `ObjectURL`，目前销毁逻辑主要依赖卡片移除。需要确保在“清空”或“刷新”时主动调用 `URL.revokeObjectURL`。

### 4. 模型同步
- 前端虽然隐藏了模型选择，但代码中仍保留了 Qwen/Kimi 等逻辑。若后端 API 变更，需要同步更新前端的 `MODEL_LABEL` 映射。
